# Audit: "Sraffa meets Polanyi — a Social Provisioning Sandbox"

**Scope.** Full audit of the uploaded `index.html` (1,179 lines): what the code claims, what it actually computes, whether the claims and explanations are precise, how the outputs should be interpreted, where the framework sits in the literature, and whether it can be developed into a rigorous research contribution. Every numerical claim below was verified by replicating the engine's exact conventions (column-normalized A, power iteration on Aᵀ, 500 iterations, L2 normalization) in Python and cross-checking against `numpy.linalg.eig`. The verification script is provided alongside this report so results can be re-run against future versions of the file.

**Headline judgment.** The mathematical core is small, mostly correct, and standard: it computes the Perron–Frobenius dominant eigenvalue and dominant left eigenvector of a column-normalized flow matrix, once for the full matrix and once for the "economic" principal submatrix. Where the tool is sound, it is sound because it reproduces well-established results (Sraffa's subsistence system, the closed Leontief model, the Kurz–Salvadori viability condition). Where it goes wrong, it goes wrong in three distinct ways: (i) **documentation–data divergence** — several documented numbers are simply false for the matrices actually shipped (Hudson, Sahlins, one preset does not run at all); (ii) **interpretive imprecision** — the central construct, "Provisioning Centrality," is described as measuring how much the system *depends on* a sector, when the left eigenvector in fact measures how much a sector *absorbs from* the system (a cost/price analogue, not a dependency measure); (iii) **a silent theoretical commitment** — closing the matrix by making surplus claimants into producing sectors quietly converts distribution into technology, which is precisely the move the surplus approach exists to reject. None of these is fatal; all three are fixable; but (ii) and (iii) go to the heart of what a paper built on this framework would have to defend.

## 1. What the code claims to do vs. what it actually does

### 1.1 The engine (verified correct in its intended domain)

The computational core does four things:

1. **Builds A** as `A[i][j] = Z[i][j] / total_output[j]` (column normalization). This matches the on-screen definition and is the standard construction of technical coefficients from a transactions matrix.
2. **Computes σ** (dominant eigenvalue) and the **dominant left eigenvector** of the full A by power iteration on Aᵀ, starting from the all-ones vector, fixed 500 iterations, Rayleigh-quotient eigenvalue estimate.
3. **Computes λ** as the dominant eigenvalue of the principal submatrix formed by non-social sectors (`np.ix_`-style row/column selection — implemented correctly).
4. **Detects "non-basic" processes** by forward reachability through Z: sector i is non-basic if, following flows i → j whenever Z[i][j] > 0, not all sectors are reachable.

All four match their descriptions **when A is nonnegative, irreducible, and primitive**. The claim in the documentation that "σ = 1.00 for the full provisioning system is a mathematical consequence of the matrix being fully closed" is *correct and provable*: if row sums equal total outputs, then Aq = q with q > 0, and for a nonnegative matrix a positive eigenvector forces the spectral radius to equal its eigenvalue, so σ = 1 exactly. (This holds even for reducible matrices, which is a nice robustness property the documentation doesn't claim but could.) Likewise λ ≤ σ always holds — the spectral radius of a principal submatrix of a nonnegative matrix never exceeds that of the full matrix — which resolves the "(TBD; a partial, lower-bound reading…?)" left dangling in the shipped caption: λ is a **lower bound on σ** and therefore an *optimistic* measure of core viability; excluding social inputs can only make the core look more viable, never less.

### 1.2 Where the engine silently fails (verified failure modes)

The power-iteration routine has no convergence check, no irreducibility/primitivity check, and no nonnegativity enforcement. Three concrete failure modes, all reachable through ordinary user editing:

**(a) Cyclic (imprimitive) matrices give wrong eigenvalues with no warning.** A two-sector pure cycle a user could easily construct — Z producing A = [[0, 2], [0.5, 0]] — has true spectral radius 1.0. The engine reports **λ = 1.25**, a 25% error, because power iteration oscillates on period-2 structures and the Rayleigh quotient stabilizes at the wrong value. Any matrix whose basic circuit has no self-loops is at risk. (All shipped presets happen to have diagonal or dense enough structure to be primitive, so this is latent, not active — but a sandbox invites users to build exactly the sparse ceremonial-cycle matrices that trigger it.)

**(b) A self-reproducing non-basic block silently hijacks σ and zeroes the weights.** This is Sraffa's "beans" problem (PCMC, Appendix B) in computational form. Test case: a basic core with ρ = 0.787 plus one non-basic self-looper with own-coefficient 1.2. The engine reports σ = 1.2 and a left eigenvector of exactly (0, 0, 1): every basic sector gets weight zero (or, if underflow doesn't reach exact zero, the normalized weights explode to ~10⁹⁰ when the numeraire sits in the core). The weights table would display this garbage with no diagnostic. Since the documentation's own discussion of non-basics ("their removal does not alter the dominant eigenvalue…of the basic sub-system") is worded to sidestep exactly this case, the *display* should detect and flag it: check whether the dominant eigenvalue of the full matrix exceeds that of the basic subsystem, and warn.

**(c) Negative entries are not prevented.** The `min="0"` HTML attribute does not stop typed or pasted negative values; `parseFloat` accepts them; Perron–Frobenius theory then no longer applies and the iteration can converge to meaningless values. Clamp on input.

Additionally, the `provisioning_weights` guard (`num_val > 0 ? … : 0`) only catches an *exactly* zero numeraire weight. Near-zero weights — the typical numerical outcome in reducible systems — pass the guard and produce astronomically scaled tables.

### 1.3 One preset does not run at all

The **Greek Oikos** preset is syntactically broken in the shipped file. Rows 3–7 of its Z contain JavaScript array *holes* (`[ 0, , 60, …]`), which evaluate to `undefined`, and `total_output` has **six entries for seven sectors**. The zero-output guard does not catch this because `undefined <= 0` is `false` in JavaScript; the missing seventh output therefore passes the check and produces `NaN` throughout A, the eigenvalue, and the weights. A preset-sanitizer (validate row/column counts, replace holes with 0, coerce to finite nonnegative numbers on load) would catch this class of error permanently.

## 3. The central interpretive problem: what the left eigenvector actually measures

The tool's flagship construct is described as follows: "A sector's provisioning centrality weight indicates the relative importance of this sector in sustaining the provisioning network, i.e., how relevant this sector is for the reproduction of all the others."

**This is not what the dominant left eigenvector of A measures, and the tool's own output proves it.** The left eigenvector solves p·A = σp, i.e., pⱼ ∝ Σᵢ pᵢA[i][j]: a sector's weight is the weighted *cost of the inputs it absorbs per unit of output*. It is the eigen-generalization of a **price** or unit-cost vector — which is exactly why it reproduces Sraffa's exchange ratios at subsistence. It is an *input-absorption* (cost-side, "authority"-side) measure. "How relevant this sector is for the reproduction of all the others" is a *forward-linkage* (supply-side, "hub"-side) question, whose natural formalizations are the dominant **right** eigenvector (which in Sraffa is the Standard-commodity proportions and in Bródy's closed system is the activity-scale dual), or row-normalized allocation structures in the tradition of Ghosh (1958) and Augusztinovics (1970).

The decisive counterexample is already in the Capitalism preset: **Waste — a terminal sink whose output sustains nothing, flagged non-basic by the tool itself — receives a Provisioning Centrality of 1.83, higher than Energy (0.96) and Work (1.00).** Under the documentation's stated reading, the system "depends" more on Waste than on Energy. Under the correct reading, Waste is simply *expensive to produce* in embodied-input terms: it absorbs weighted inputs and returns nothing. The number is fine; the sentence describing it is wrong. The same slippage appears in the radar chart's title ("Systemic Reproduction Weights") and the A-network caption ("higher systemic centrality per unit of output").

Three consequences:

1. **Rename or re-describe.** Either describe the left eigenvector honestly ("eigen-cost / embodied-input weight; the generalization of Sraffa's subsistence prices"), or compute what the current description promises (the right eigenvector, displayed as a second, dual panel). The genuinely attractive move — with direct precedent in Bródy (1970), on which more below — is to display **both** and teach the duality: left = valuation/cost, right = scale/dependency. That duality *is* the classical price–quantity duality, and making it visible would be a real pedagogical contribution.
2. **"Centrality" is a loaded borrow.** In network science, eigenvector centrality of a flow graph has established variants (Bonacich, Katz, hub/authority), and the left-eigenvector-of-A construction corresponds to a specific one of them. A reviewer from the Acemoglu et al. (2012) production-networks tradition — which the reference list already invokes — will note that their "influence vector" is a Leontief-inverse-based object, not a Perron vector of A, and will expect the relationship to be stated. Precision here is cheap and the current vagueness is expensive.
3. **The "analogue to prices" hedge is closer to the truth than the "importance" claim.** The caption's "(in some sense to be explored) analogues to prices" is right; the intro sentence above it is wrong. The document currently asserts both.

A related but distinct precision point: the weights are **unit-dependent in an exactly characterizable way**. Rescaling sector k's physical unit by factor c transforms A by a similarity D·A·D⁻¹, leaving both eigenvalues *and every value-flow chart* (Gross Sectoral Scale, Absorption, Net Flow, Institutional Breakdown — all built from Z[i][j]·wᵢ or wᵢqᵢ, which are invariant) unchanged, while multiplying the displayed weight of sector k by 1/c. So the weights table and the radar profile are the *only* unit-dependent displays, and their shape is partly an artifact of unit choices ("actions," "contracts," "events" are not units at all). The intro to the radar chart states this correctly ("sensitive to scaling units…but not to choice of numéraire"), but the caption directly beneath it states the opposite ("Axes are not comparable across different numéraire choices"). The caption is wrong; numeraire choice rescales all weights uniformly. Fix the caption, and consider stating the invariance result explicitly — it is a genuinely useful theorem for users: *eigenvalues and value-flow charts are measurement-convention-proof; raw weight profiles are not.*

## 4. The central theoretical problem: closure converts distribution into technology

This is the deepest issue and the one on which any paper stands or falls.

### 4.1 What closing the matrix does

Sraffa's Chapter 2 point — reproduced correctly in the tool's own Section 1 — is that once there is a surplus, the physical system is one equation short: distribution must come from outside (the wage, or Sraffa's later monetary-rate-of-interest closure via Pivetti). Garegnani's "core" formalizes this: quantities, technique, and one distributive variable are data; prices and the other distributive variable are determined *given* the institutional closure — under capitalism, competition enforcing a uniform rate of profits.

The sandbox closes the system differently: it promotes every claimant on the surplus (Workers, Capitalists, Temple, F.I.R.E.) to a *producing sector* with an output ("hours," "actions," "contracts") that re-enters production as an input. Formally this is the **closed Leontief model** (Leontief 1941; the household-as-industry construction), whose canonical Marxian statement is **Bródy (1970), *Proportions, Prices and Planning***: close the matrix, the dominant eigenvalue is 1, prices are the left Perron vector, quantities the right. The math is impeccable and the σ = 1 result is exactly Bródy's.

But the *economics* of that closure is not innocent, and the Sraffa Ch. 2 preset demonstrates it numerically. Three price systems now coexist for the "same" economy:

- Sraffa's subsistence ratios (Ch. 1): p_iron/p_wheat = **10**;
- Sraffa's surplus prices at the uniform r = 25% (Ch. 2): p_iron/p_wheat = **15**;
- the sandbox's closed-matrix eigen-weights for its Ch. 2 preset: w_iron/w_wheat = **20.73**.

The tool displays only the third and the documentation does not mention the second. The 20.73 is not a price "at one point on the wage–profit curve," as the preset text implies; it is the eigen-cost vector of a *different economy* — one in which capitalist consumption has become a technical input requirement (Capitalists "supply" 60 hours to wheat and 40 to iron) and the surplus wage has become part of workers' reproduction technology. **The wage–profit trade-off — the single most important object in PCMC — is unrepresentable in the current framework**, because the degree of freedom it lives on has been spent on closing the matrix.

Worse, from the standpoint of the tradition the tool claims: making the elite's consumption an input for which the elite's "output" is technically required is formally the marginal-productivity move — profit as payment to a productive factor — which the surplus approach defines itself against. The Cesaratto working paper that anchors this project makes the countervailing structure explicit in its grain–coal tributary example: the elite is a *claimant* on a surplus produced by others; exchange ratios in pre-capitalist formations are determined *once the material composition of the surplus is fixed*, and they change when that composition changes; only capitalist competition (uniform r) makes prices composition-independent. Notably, the sandbox's closed-matrix weights are *also* composition-dependent — change how the 175 surplus wheat is allocated and the weights move — so the tool accidentally exhibits a result homologous to Cesaratto's. But it produces it by the theoretically wrong route (claimants as producers) rather than the right one (claimants as claimants).

### 4.2 What to do about it

There is a clean fix that is *more* faithful to the surplus approach and requires only a modeling convention plus a UI toggle: **represent pure claimants as sinks — populated columns, zero (or near-zero) rows — and leave the system open.** Then: the economic core's λ is unchanged; surplus appears as flows into sink columns; claimants get well-defined eigen-costs (like Waste) without being imputed productivity; the balance table shows exactly which physical goods the claims absorb; and closure becomes an explicit, user-visible *choice* between (a) open-with-claimant-sinks (the surplus-approach representation), (b) closed-with-endogenized-reproduction (the Leontief–Bródy representation, appropriate when the "social sector" genuinely produces a necessary input — labor via care work being the strongest case, per Picchio), and (c) intermediate cases. The theoretically interesting question — *which* sectors belong on which side, i.e., what counts as necessary social reproduction versus surplus absorption — then stops being smuggled in through preset design and becomes the explicit analytic axis of the framework. That question is, in substance, the substantivist reformulation of the wage boundary (subsistence vs. surplus wage) that Sraffa flags in §8 of PCMC, and it is a question a paper can actually be about.

The current documentation half-sees this ("establishing more carefully the meanings of 'surplus' in a substantivist view in which the matrix is closed…"), but frames it as future reflection while the tool's presets have already answered it silently, four different ways, in four presets.

### 4.3 A redundancy the closure creates (provable, and verified)

The "Net Flow Position of Social Sectors" chart computes absorbed − contributed = Σᵢ Z[i][s]·wᵢ − wₛ·Σⱼ Z[s][j]. By the eigen-equation, absorbed = σ·wₛ·qₛ; if the matrix is closed, contributed = wₛ·qₛ and σ = 1, so **net flow ≡ 0 for every sector in every closed matrix** — verified to numerical precision in Sraffa Ch. 2, both closed Quesnay presets, Greek Polis, Roman Latifundium, and Simon (all nets = ±0.000). The chart therefore displays rounding noise in exactly the presets showcased as closed, and displays w_s(σqₛ − rowsumₛ) — i.e., a price-weighted restatement of the balance table's row imbalances — otherwise. The caption's worry about "different weights…creating an asymmetry that should be interpreted with care" misdiagnoses the situation twice over: valuing each flow at the price of the good flowing is the *only consistent* valuation (there is no asymmetry problem), and the real issue is that the statistic is an accounting identity, zero by construction under closure. Similarly, the "Gross Input Absorption" percentages reduce, at σ = 1, to each social sector's share of total value — the same information as the Gross Sectoral Scale chart. Recommendation: either drop these two charts, or redesign them around quantities that are *not* pinned by the eigen-identity (e.g., decompose each social sector's absorbed value by *source* sector or by institutional annotation — that decomposition is not determined by the identity and is genuinely informative).

## 5. Other interpretive and precision issues, in decreasing order of weight

**σ, the balance table, and the Capitalism preset's ecology claim.** σ is a property of the *coefficient structure* (columns); the balance table is a property of *levels* (rows). The documentation distinguishes them correctly in one caption, then blurs them in the Capitalism text: σ = 0.985 < 1 is glossed as "the complete system is able to reproduce itself (at least while ecological stocks are not depleted)" — but the Biosphere row is in deficit by 300 (extraction 1,100 against regeneration 800), so at current levels the system is running down stocks *now*, not "at least while." What σ < 1 licenses is a statement about the coefficient structure admitting proportional viability, not about the sustainability of the configured levels. The λ/σ contrast is the tool's best idea (see §7); it deserves exact language, and the exact language is available: *λ measures the core's viability treating social and ecological inputs as free; σ measures the full structure's viability; the balance table measures whether the currently configured levels are mutually consistent. Three different questions.*

**Non-basic detection: right rule, two caveats.** Forward reachability on the support of Z is the correct graph-theoretic reading of Sraffa's definition, and the "Feasts are basic" result in Sahlins is genuine and well-used in the text. Caveats: (a) the routine seeds the visited set with i itself, so it never checks whether a sector enters (even indirectly) its *own* production — harmless in practice, worth a code comment; (b) the beans problem of §1.2(b) means the *displayed* σ and weights can be dominated by a non-basic block, in which case the reassuring parenthetical in the UI ("removal does not alter the dominant eigenvalue…of the basic sub-system") is technically true but misleading about what is on screen. Detect and warn.

**λ = 0 display.** Acyclic economic cores (Simon Firm) produce λ = 0, rendered as "Economic subsystem generates a surplus." A nilpotent core has no internal reproduction circuit at all; "surplus" is the wrong word. Special-case the message.

**Numeraire caption.** "The numéraire choice only rescales quantities of other sectors" — it rescales *weights*, not quantities. Also, since the numeraire radio is user-movable onto a sector whose eigen-weight is zero or near-zero (reducible cases), the normalization needs the guard discussed in §1.2.

**Silver–grain peg (Hudson).** As implemented, the identical per-unit input columns make 1 shekel exchange-equivalent to 1 *sila*, not 1 *gur* (300 sila). The historical texture in the prose (60 sila monthly rations; 1 shekel ≈ 1 gur ≈ 300 sila) is consistent with the standard literature (the ration figures match Gelb's classic account of the Mesopotamian ration system; the shekel–gur equivalence matches Old Babylonian norms), which makes the 300× implementation error more visible, not less. Multiply the Silver column's coefficients by 300 (or divide its output unit), and re-verify.

**Threshold bands.** The surplus/subsistence/deficit banners switch at 0.99/1.01. A system at λ = 0.995 reads "subsistence." Harmless pedagogically, but say so ("displayed classification uses a ±1% band"), because a user replicating the number in another tool will get a different verbal verdict.

**"5 preset matrixes."** There are five *named buttons* plus Custom plus seven dropdown presets — twelve matrices. Also "matrices."

**Sector count / naming mismatches.** "Industry and Services" (Capitalism text) — no Services sector exists. "400 hours…to the Foraging sector" (Sahlins text) — the data allocates 600 to Foraging and 400 to Feasts.

**Language.** The shipped prose contains at least the following typos/malformations, several in reviewer-visible load-bearing paragraphs: "In factnon-market", "incuding", "tothrough", "waves towards" (presumably "gestures towards"), "sectoprs", "towrads", "attemp", "numéraira", "we have collapse" (→ collapsed), "This does affects", "is a parameters", "should be understood are", "indicating a that the complete system", "Polayian" (Technical Dependency Network caption), "occuring", lowercase "sraffa's", and the stray "(TBD…?)" inside a public-facing caption. Also "adults would roughly 12–19 hours per week" is missing its verb.

**Minor engineering.** `removeSector` only removes the *last* sector (cannot delete from the middle); chart.js and vis-network load from CDNs, so the file is not actually self-contained offline; the stale-button pattern is good UX and worth keeping.

## 6. Has this been done before? Positioning in the literature

The honest answer is: **the mathematics is old; the specific synthesis and the diagnostic use are not, so far as I could establish.** In detail:

**Direct formal precedents (must engage; currently uncited).**
- **Leontief's closed model** (households endogenized as an industry, 1940s) is the exact formal operation the tool performs when it "adds social sectors and closes the matrix." Any reviewer in the input-output tradition will name it in the first paragraph of their report.
- **Bródy (1970), *Proportions, Prices and Planning*** is the single closest ancestor: a closed Marx–Leontief system in which the maximal eigenvalue equals one and prices and quantities are the dual left/right Perron eigenvectors. The tool's σ = 1 result, its left-eigenvector "weights," and the duality it currently *fails* to display are all in Bródy. This is not a threat — it is a foundation: citing Bródy converts "non-rigorous extension" into "an interactive implementation of a known closed-system formalism, extended in a Polanyian direction." There is also a follow-on empirical literature on eigenvalue spectra of real input-output tables (Bródy's 1997 conjecture and its Sraffian discussion in the Mariolis–Tsoulfidis line of work) that becomes relevant the moment calibration starts.
- **von Neumann (1937/1945)** for the growth-factor/interest-factor duality (the deeper reading of σ as an expansion factor), best accessed through **Kurz & Salvadori (1995)**, whose textbook definition of *viability* (spectral radius ≤ 1) is precisely the tool's λ-criterion — a citation that would anchor the tool's central banner in the standard reference.
- **Ghosh (1958) / Augusztinovics (1970)** for forward-linkage (supply-side) measures — the thing the "Provisioning Centrality" prose currently *claims* to measure.

**Adjacent traditions the reference list already gestures at, correctly.** Hannon (1973) and the subsequent ecological network analysis literature (Finn's cycling index, Patten's environ analysis, Ulanowicz) contain the closest analogue to the λ/σ move: comparing a subsystem's throughput structure to the full system's. The physical input–output table (PIOT) literature contains a *directly relevant controversy* about how to treat waste and sinks in physically-closed accounting (the Giljum–Hubacek vs. Suh exchange in the mid-2000s) — the tool's Waste and Biosphere conventions should be positioned relative to it. Time-use-extended input–output and SAM work (Duchin on household lifestyles; Antonopoulos & Hirway on unpaid work; the Pyatt–Round SAM tradition already cited) is the precedent for the Work sector's unpaid-care self-loop. Hayden's Social Fabric Matrix is the institutionalist cousin — a matrix of deliveries between social and economic components — but it is qualitative/Boolean in spirit and has no reproduction eigenstructure, which is exactly the gap this tool occupies relative to it.

**The Polanyi side.** I could not find any prior work that formally couples Polanyi's forms of integration to a reproduction matrix. The quantitative Polanyi literature (Pryor's cross-cultural statistical tests of the modes of integration in the 1970s) is statistical, not structural. Cesaratto (2024 JEI; the 2026 STOREP working paper; Cesaratto & Di Bucchianico 2021a,b) supplies the *theoretical* bridge — surplus approach as the "third way" between formalism and substantivism, with worked numerical examples of tributary-economy exchange ratios — but no matrix formalism, no eigenstructure, no tool. **The niche is real: an interactive, eigen-structural operationalization of the surplus-approach-meets-substantivism program does not, to the best of my searching, exist.** The corollary is that the Polanyian layer is currently the tool's *least* developed part (inert annotations), which means the claimed novelty and the implemented novelty do not yet coincide.

**What is genuinely novel, if made rigorous.** Three candidates, in descending order of confidence: (1) the **λ/σ pair as a quantitative embeddedness diagnostic** — "how much of the commodity core's apparent surplus depends on pricing social and ecological inputs at zero" is a crisp, computable, unit-invariant statement of the fictitious-commodities argument, and I found no prior formulation of it in these terms; (2) the **claimants-as-sinks vs. claimants-as-producers closure taxonomy** of §4.2, which would give the "meanings of surplus in a substantivist view" question an exact formal skeleton and connects directly to the exchange-ratio results in the Cesaratto working paper; (3) the tool itself as **pedagogy** — there is no comparable interactive teaching instrument for Sraffian reproduction logic applied beyond capitalism, and heterodox teaching materials are a recognized genre in JEI/AJES-adjacent venues.

## 7. Can this support rigorous research claims? A frank assessment and a path

**As it stands: no**, and the documentation says so itself, correctly and repeatedly — the "intuition pump" framing, the disclaimers on presets, and the "annotations only" honesty about the institutional layer are the best-written parts of the document and should be preserved verbatim through any revision. The specific gaps between here and research-grade are: uncalibrated presets with documentation–data divergences; an engine without convergence/irreducibility diagnostics; a flagship construct described incorrectly; a closure convention with unexamined theoretical content; and an institutional layer that does no analytic work.

**But the distance is shorter than it looks**, because the deficiencies sort cleanly into (a) engineering hygiene (sanitizer, regression harness, warnings, clamps — days of work), (b) rewriting descriptions to match the mathematics (the left/right eigenvector fix, the σ/balance distinction, the caption corrections — days), and (c) genuine theory (the closure question, the operationalization of integration forms — this is the paper, not the tool).

A realistic development path, consistent with a methods-paper-plus-case-study program:

**Stage 1 — make the tool tell the truth about itself.** Fix the broken and mis-documented presets; add the regression harness; add the three warnings (imprimitivity/non-convergence, non-basic-dominated eigenvalue, near-zero numeraire weight); clamp negatives; display the right eigenvector next to the left and describe both correctly; remove or redesign the two redundant charts; resolve every item in §5. After this, the tool is a *sound* intuition pump — publishable as pedagogy in its own right (JEI and AJES both carry teaching-oriented pieces; a short "computational appendix" genre also exists in *Review of Political Economy* / *Metroeconomica* space).

**Stage 2 — make the closure question explicit and central.** Implement the open-with-claimant-sinks mode; let users toggle each social sector between "claimant (sink)" and "reproductive (producing)" status; show how λ, σ, and the weights respond. This turns the tool's biggest current liability into its research instrument: the toggle *is* the substantivist surplus question, and the sensitivity of the weight vector to the toggle *is* the formal content of "prices presuppose an institutional closure." This stage is where engagement with the Garegnani-core problem becomes concrete rather than gestural: the framework must say what, in a non-capitalist formation, plays the determinacy-providing role that uniform-r competition plays in the core — and the Cesaratto working paper's composition-dependence result suggests the answer (the institutionally fixed composition and destination of the surplus) is representable in exactly this apparatus.

**Stage 3 — one calibrated case.** The Mesopotamian ration-list material (Hudson & Wunsch; the standard ration literature) is unusually well suited: rations are literally administered input coefficients, which sidesteps the worst identification problems of imputing coefficients to non-market economies. A single calibrated palace-economy matrix, with explicit sourcing per cell, uncertainty ranges, and the λ/σ diagnostic computed honestly, would convert the framework from illustration to instrument — at postdoc scale, exactly one such case is enough.

**Standing risks to manage.** Overclaiming from stylized matrices (the register throughout should remain "this formalism *represents* claims; it does not *establish* them"); the "centrality" terminology collision (§3); the temptation to let the eigenmath run ahead of institutional meaning — the internal test worth adopting for every displayed statistic is the one already implicit in the best captions: *what institutionally real thing does this number represent, and would it survive a change of units, of numeraire, and of closure convention?* Of the tool's current displays, exactly three pass all parts of that test today: λ, σ, and the value-flow aggregates. That is a perfectly good foundation. It is just smaller than the interface currently implies.
